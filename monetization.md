# Монетизация: за что платят, экономика, платёжка

Отчёт по ценности продукта, себестоимости, каналам оплаты и техническим блокерам.
Цены актуальны на июль 2026: Gemini 2.5 Flash — $0.30/M input, $2.50/M output;
выплата по Telegram Stars — ~$0.013/звезда (десктоп) / ~$0.009 (мобильные,
комиссия Apple/Google 30%), блендированно **~$0.010/звезда**.

## 1. За что платит пользователь

«Доступ к LLM» не продаётся — это бесплатно есть в ChatGPT/Gemini. Платят за то,
чего у голой LLM нет:

1. **Персональная память** — агент знает профиль, дневник, цели, историю диалогов.
   Накопленный контекст = switching cost, фундамент retention.
2. **RAG-обоснованность** ([RAG.md](RAG.md)) — ответы с опорой на реальные фрагменты
   первоисточников («Размышления, книга IV»), а не пересказ LLM. Осязаемый
   премиум-дифференциатор при почти нулевой себестоимости (см. §2).
3. **Проактивность** — настраиваемые уведомления: утренний стоик-брифинг, вечерняя
   рефлексия, своё время и частота. Возвращает пользователя ежедневно — это и
   ценность premium, и канал конверсии free-пользователей.
4. **Лимиты сообщений** — механика пейволла поверх первых трёх.

### Матрица Free / Premium

| | Free | Premium |
|---|---|---|
| Сообщения агентам | 10/день | 50/день или безлимит |
| Ответы | обычные | RAG: с цитатами и указанием источника |
| Уведомления | 1 напоминание, фикс. время | настраиваемые + брифинги от агентов |
| Цели | 1 активная | несколько |
| Память диалога | короткая (текущие 4 реплики) | расширенная |
| Разборы | — | недельный разбор дневника |

## 2. Себестоимость одного сообщения

Состав запроса (`app/services/agent_chat.py`):

- input ≈ **1 500 токенов**: системный промпт ~500 + профиль/история (4×700 симв.)/дневник ~1 000;
- output ≈ **350 токенов** ответа + thinking-токены (биллятся как output).

| Конфигурация | Стоимость сообщения |
|---|---|
| Сейчас (`gemini-2.5-flash`, thinking включён по умолчанию) | **~$0.003** |
| thinking = 0 (рекомендуется) | **~$0.0013** |
| Gemini 2.5 Flash-Lite ($0.10/$0.40) | ~$0.0003 |

Добавки:

- continuation/retry-вызовы (`_complete_if_needed`, `_retry`) — ~+10%;
- **RAG**: +400–700 input-токенов цитат ≈ **+$0.0002/сообщение**; embedding вопроса —
  копейки ($0.15/M); разовая индексация корпуса ~3 тыс. чанков ≈ **$0.15**.
  RAG практически не влияет на себестоимость;
- **уведомления с LLM-текстом**: 1 генерация ≈ $0.001 → ежедневный брифинг
  ≈ $0.03–0.04/мес на пользователя.

## 3. Расход на пользователя в месяц (при thinking = 0)

| Профиль | Сообщений/мес | LLM-расход |
|---|---|---|
| Случайный | 30 | ~$0.04 |
| Активный | 150 | ~$0.20 |
| Тяжёлый | 500 | ~$0.65 |

Инфраструктура фиксированная: Railway (app + Postgres + Redis) ≈ **$15–20/мес**,
Vercel — бесплатно. Не растёт до нескольких тысяч пользователей.

## 4. Каналы оплаты

### Правило Telegram

Цифровые товары и услуги **внутри Telegram-приложений продаются только за Stars**
(требование политик Apple 3.1.1 / Google Payments). Провайдеры из BotFather
(Robokassa KZ, ЮKassa, Freedom Pay и т.д.) — Bot Payments API — официально
**только для физических товаров и услуг**. Подписка на бота через ЮKassa внутри
бота = нарушение ToS: бот/Mini App могут скрыть от мобильных пользователей.

Ограничение действует именно *внутри* Telegram: оплата на **внешней странице**
(свой сайт + платёжный шлюз) под него не попадает — стандартная практика
подписочных ботов.

### Сравнение каналов (подписка ~$6/мес)

| Канал | Нетто вам | Плюсы | Минусы |
|---|---|---|---|
| **Stars, 299/мес** | **~$3.0** | полное соответствие правилам, работает в Mini App, не нужно юрлицо | комиссия сторов ~30% (моб.), вывод: холд 21 день, мин. 1000 звёзд, через Fragment в TON |
| **Карта на внешней странице** (Robokassa KZ / Freedom Pay / CloudPayments, ~3–4%) | **~$5.7** | почти вдвое больше нетто, выплаты в тенге на счёт, без холда | нужен ИП, ссылка наружу из бота — серая зона, конверсия ниже (лишний шаг) |

### Рекомендация: гибрид

1. **Stars в Mini App** — основной канал, соответствующий правилам
   (`sendInvoice` с `currency="XTR"` / `openInvoice`).
2. **Карта на внешней странице** — параллельно, со скидкой
   («на сайте дешевле») — так офсетят комиссию сторов почти все подписочные боты.
3. Оба канала активируют один и тот же `plan/premium_until` через вебхуки сервера.

## 5. Сходимость

- **Маржа с подписчика:** Stars: $3.0 − $0.2–0.65 LLM = **$2.4–2.8 (80–90%)**;
  карта: $5.7 − LLM = **$5.1–5.5 (~90%)**.
  С включённым thinking тяжёлый пользователь съедает $1.5+ — маржа падает вдвое.
- **Безубыточность по инфре ($20/мес):** ~7 подписчиков через Stars, ~4 через карту.
- **1 000 MAU, конверсия 3% (30 подписок):**
  - только Stars: доход $90; расходы free ~$25 (thinking = 0) + инфра $20 →
    **~$45/мес прибыли**;
  - гибрид (50/50 Stars/карта): доход ~$130 → **~$85/мес**;
  - при текущей конфигурации (thinking включён) free-расход ~$70 → около нуля.
  Дальше линейно с конверсией и MAU.

## 6. Технические блокеры перед запуском

По убыванию важности:

- [ ] **Thinking выключить**: `"thinkingConfig": {"thinkingBudget": 0}` в
      `_request_body` (`agent_chat.py`). Латентность первого текста −60%
      (с 2–4 с до ~1 с), себестоимость −65%. Главный рычаг по скорости и деньгам.
- [ ] **Квоты**: `tokens` нигде не декрементируется, `plan` не проверяется —
      сейчас любой пользователь = неограниченные вызовы Gemini. Проверка и
      списание перед каждым вызовом + анти-флуд (запрет параллельных генераций
      на пользователя).
- [ ] **Дыра в `PATCH /api/me`**: `ProfileUpdate.plan` (`api/schemas.py`)
      редактируется клиентом — пользователь может сам выставить себе Premium.
      Убрать из схемы; `plan` выставляет только сервер после оплаты.
- [ ] **Переиспользуемый `httpx.AsyncClient`** в `clients/gemini.py`: сейчас новый
      клиент на каждый вызов = полный TLS handshake (~100–300 мс) на каждое сообщение.
- [ ] **Интервал стрим-редактирования** → ~1 с (`chat.py`, сейчас 0.25 с):
      лимит Telegram ~1 edit/сек на чат, излишки уходят в flood-wait впустую.
- [ ] Мелкое: залипание `_redis_failed` (история отключается до рестарта при
      моргнувшем Redis); rate-limit Gemini — free tier всего 10 RPM, для прода
      нужен платный tier.

## 7. Реализация оплаты (следующий шаг)

1. Миграция: `plan` → enum (`basic|premium`), поле `premium_until: datetime | None`,
   `tokens` → дневной счётчик с датой сброса.
2. Stars: `sendInvoice` с `currency="XTR"`, обработчики `pre_checkout_query` и
   `successful_payment` → сервер выставляет `plan/premium_until`.
3. Mini App: кнопка апгрейда → `Telegram.WebApp.openInvoice`; кабинет читает
   `plan/tokens` как сейчас.
4. Внешняя страница: лендинг с оплатой (Robokassa KZ / Freedom Pay), вебхук
   провайдера → тот же серверный код активации по Telegram ID.
5. При исчерпании квоты — сообщение с кнопками покупки вместо вызова Gemini.
