# Маркетинг: путь до 10 / 100 / 1000 активных пользователей

Стратегия привлечения для aeon. Связана с [monetization.md](monetization.md):
экономика сходится при CAC ≤ $1/MAU и удержании D7 ≥ 30%.

**«Активный пользователь» (MAU/WAU)** = написал агенту хотя бы одно сообщение
за последние 7 дней. Воронка, которую меряем с первого дня:

```
источник (start-параметр) → /start → онбординг завершён → 1-е сообщение агенту
                                                        → возврат на 7-й день
```

Три этапа — три разные игры: до 100 пользователей это не «маркетинг»,
а ручная работа и подготовка продукта.

## Этап 0 → 10: руками (~1 неделя)

Цель — не пользователи, а обратная связь и калибровка воронки.

- [ ] 10–20 личных сообщений знакомым из целевой аудитории (саморазвитие, книги,
      предприниматели). Формулировка: не «потести бота», а «выбери агента и задай
      реальный вопрос, который тебя грызёт».
- [ ] Смотреть на поведение, не на похвалу: дошёл до конца онбординга? написал
      второе сообщение? вернулся на следующий день?
- [ ] Определить агента-«лицо продукта» (гипотеза: Макиавелли — самый острый
      и шеримый). Он пойдёт в заголовки постов и роликов.

## Этап 10 → 100: комьюнити и бесплатные площадки (1–2 месяца)

Бот двуязычный — работаем на два рынка сразу.

**RU:**
- [ ] Посевы у микроканалов (1–10 тыс. подписчиков) в нишах: стоицизм,
      саморазвитие, книги, психология. Бесплатно / бартер / символически.
- [ ] Каталоги ботов (@BotsArchive и аналоги), каталог Telegram Mini Apps.
- [ ] Кейс-статья на vc.ru / Habr: «Как я сделал ИИ-наставника-стоика
      в Telegram» — технические кейсы стабильно дают первые сотни установок.

**EN:**
- [ ] Reddit: r/Stoicism (~полмиллиона), r/Jung, r/TelegramBots. Не реклама,
      а пост-история: «I built a bot to talk to Marcus Aurelius, grounded in
      the actual Meditations». RAG — главный аргумент против
      «очередной GPT-обёртки», поэтому RAG стоит доделать до этого этапа.
- [ ] Product Hunt — разовый всплеск трафика + бэклинки.

## Этап 100 → 1000: виральные механики + платные посевы (3–6 месяцев)

### Сначала вшить в продукт (снижает CAC на всё последующее)

- [ ] **Шеримый memento mori**: генерация картинки «твоя жизнь в неделях»
      с подписью бота и кнопкой «поделиться». Самый вирусный формат ниши —
      люди сами постят его в сторис.
- [ ] **Реферальная программа**: `?start=ref_<telegram_id>` — «+20 сообщений
      тебе и другу». Дешевле любой рекламы; поле `tokens` уже есть.
- [ ] **Шеринг ответов агента**: кнопка «переслать» под ответом — каждый
      пересланный ответ = реклама в чужом чате.

### Потом платное

- [ ] **Посевы в TG-каналах** (RU: саморазвитие/книги/психология) — главный
      масштабируемый канал. Ориентиры: ~$0.1–0.3 за `/start`, при удержании
      30–40% → **$0.3–1.0 за MAU** → бюджет до 1000 MAU ≈ **$300–700**.
      Начинать с 3–5 тестовых каналов по $20–50, масштабировать те, где
      CAC ниже (поэтому нужны start-метки по каналам).
- [ ] **Короткие видео** (TikTok/Reels/Shorts): ниша стоицизма огромна и дешева.
      Формат — скринкаст диалога: «спросил у Макиавелли, как поступить
      с токсичным начальником». Один залетевший ролик = сотни установок бесплатно.
- [ ] Telegram Ads — только после посевов: порог бюджета выше, таргетинг хуже
      ручного выбора каналов.

## Сходимость с экономикой

| Параметр | Значение |
|---|---|
| CAC (посевы) | ~$0.3–1.0 за MAU |
| Конверсия в premium | ~3% → ~$15 за подписчика |
| Доход с подписчика | $3.0 (Stars) – $5.7 (карта) в мес |
| Окупаемость подписчика | 3–5 месяцев |

Два условия до запуска платного трафика (иначе льём воду в дырявое ведро):

1. **D7 retention ≥ 30%** — проверить на первых 100 органических пользователях;
2. **квоты + thinking = 0** включены (см. блокеры в
   [monetization.md](monetization.md), §6) — иначе каждый привлечённый
   бесплатник стоит втрое дороже.

## Аналитика (минимум, с первого дня)

- [ ] `start`-параметр источника → колонка `acquired_from` в `users`.
- [ ] События: онбординг завершён, 1-е сообщение, N-е сообщение, возврат D1/D7.
      Достаточно полей/таблицы в Postgres + SQL-запросов, сторонняя аналитика
      не нужна.
- [ ] Еженедельный срез: /start по источникам → конверсия в 1-е сообщение →
      D7 → конверсия в оплату.
